> 모든 월간 리포트는 이 양식을 따른다. 새 리포트 만들 때 이 문서 + 해당 채널 `_context.md`를 먼저 읽는다.
> 시작 템플릿: **성과형 = `mun-wonjang/`** · **진단형 = `hwang-wonjang/`** 의 최신 HTML을 복사해 숫자만 교체.

## 공통 골격 (위=일반 → 아래=상세, 9파트)

1. **헤더** — 채널명 · 기간 · 구독자/누적조회
2. **진행중 노트** — 월 중간이면 “하루평균 환산” 명시 (노란 박스)
3. **한 줄 평** — “이번 달 잘했나” Trunk Test 답. 흰배경+좌측 초록띠. 문장별 줄바꿈(점머리). 2~3줄.
4. **채널 전체 스냅샷** — 하루평균. 전월대비 ▲▼색. (조회·구독·노출·시청)
5. **형식별** — ⚡쇼츠(보라) / 🎬롱폼(초록) 분리. 끝까지 본 비율 포함.
6. **[변종]** — 성과형 → 에버그린 블록 / 진단형 → 심층 진단
7. **영상** — 이달의 영상(성과형) / 해답 힌트(진단형)
8. **다음 달 할 일** — 3~5개. ★ = 원장 결정/실행
9. **부록(접힘)** — 분석 기준·분류·데이터 출처

## 2 변종 (⑥⑦ 갈림)

| | 성과형 (문원장) | 진단형 (황원장) |
|---|---|---|
| 언제 | 고민 없음, 잘 됨 | 원장 고민 있음 |

| ⑦ | 이달의 영상(베스트) | 해답 힌트(모범 영상) |

## 불변식 (모든 리포트 공통)

1. **하루평균 환산** — 월 중간이면 날짜수 다른 달과 공정 비교 위해 전부 ÷일수.
2. **쇼츠/롱폼 분리** — 섞으면 거짓말. 길이 90초 이하=쇼츠, 초과=롱폼.
3. **원장친화 용어** — 평균조회율 → “끝까지 본 비율”, CTR → “썸네일 클릭률”.
4. **한줄평 문장별 줄바꿈** (점머리, run-on 금지).
5. **색**: 롱폼=초록(`#0b7a5e`) / 쇼츠=보라(`#7b3fe4`) / 진단=주황(`#c2410c`).
6. **KPI = 내원** — 조회 ≠ 내원 명시. YouTube가 보상하는 영상이 내원 영상이 아닐 수 있음.
7. **noindex** (검색 차단) + 자체완결 HTML (CSS 내장, 링크 깨짐 방지).
8. 매달 **`_context.md` 먼저 읽고** → 새 CSV 집계 → baseline·내원로그 갱신 → 리포트 생성 → push → 링크.

## 핵심 분석 원리 (진단형에서 검증됨)

- **조회 ≠ 내원.** 초기·미진단 증상 영상은 조회 높지만 대학병원 감. 고가치 환자(진단됨+양약실패+절박)는 검색파이 작아 조회 낮음. → 의도적으로 조회 낮은 절망 콘텐츠 집중.
- **검색 채널** = 질환별 검색 파이(노출/편)가 도달 결정. 카피·추천보다 검색수요가 1차.
- **타겟 질환 4기준** = ①심각도 floor 높음(단순 케이스 없음) ②양방 한계 명확 ③한방 행동 문화 ④검색 풀 적정. floor 높은 질환 = 이름 자체가 절박 환자 필터.

## 데이터 인테이크 (YouTube Studio)

- **표 데이터.csv** (영상별: 길이·조회·시청시간·구독·노출·CTR) — 핵심
- **총계.csv** (일별 조회) — 같은창 비교용
- 차트 데이터.csv = 불필요
- 기간: 이번달 + 지난달 (+ 첫 채널 1회 누적 전체)
- 집계는 `aggregate.py` 사용.

## 매달 작업 흐름

1. `_format.md` + `[채널]/_context.md` 읽기
2. YouTube Studio CSV 받기 (표+총계, 이번달·지난달)
3. `python3 aggregate.py` 로 집계 (하루평균·형식split·끝까지·카테고리·에버그린)
4. 변종 판단 (고민 있으면 진단형) → HTML 채움
5. `_context.md` baseline·내원로그·히스토리 갱신
6. commit + push → 1~2분 후 라이브 → 링크 전달
